## Documentation Standards

All documentation follows these standards:

### Structure

- Clear headings and sections
- Table of contents for long documents
- Code examples with syntax highlighting
- Links to related documentation

### Code Examples

- Complete, runnable examples
- TypeScript with proper typing
- Comments explaining key concepts
- Real-world use cases

### Writing Style

- Clear, concise language
- Active voice
- Step-by-step instructions
- Practical examples

## Contributing to Documentation

When updating documentation:

1. **Keep it current:** Update docs with code changes
2. **Be clear:** Write for your audience's skill level
3. **Add examples:** Show, don't just tell
4. **Link related docs:** Help readers find more info
5. **Test examples:** Ensure code examples work
6. **Follow format:** Match existing documentation style
